Guia rápido

Requisitos:

  • PHP 5.3 até 5.5;
  • Symfony 2.3.21 ou superior;
  • Apache - recomendada a versão 2.4.6;
  • PostgreSQL.

Instalação:

PHP 5 e Apache 2

A instalação de ambos pode ser realizada tanto no linux quanto no windows.

- Windows: É recomendado a instalação do Wamp ou Xamp.
- Linux: Pode-se executar o seguinte comando via terminal: sudo apt-get install apache2 php5.

Após a instalação, é necessário configurar o apache de modo que ele trabalhe com o .htaccess.

- Linux: Basta digitar o comando: sudo a2enmod rewrite
- Windows (WAMP): Dentro do arquivo httpd.conf, procure pela palavra rewrite_module; provavelmente irá encontrar #LoadModule rewrite_module modules/mod_rewrite.so; Apenas remova o #, e salve o arquivo.

No arquivo php.ini descomente a diretiva "extension=fileinfo.so"


Caso esteja utilizando Windows, deve-se configurar a variável de ambiente para utilizar o comando do PHP.
Para realizar esse procedimento, caso esteja utilizando Wamp, basta utilizar este vídeo como referência de configuração: https://www.youtube.com/watch?v=jB5TvzzggWw ( Créditos: Bruno Eduardo ).

Pronto! Teóricamente está tudo pronto para começar a instalação do Symfony2.

Symfony2.4.4

Depois de realizar a instação do apache e php, pode-se iniciar a instalação do Symfony2.

O próprio site do Symfony2 possui um local que ensina a instalar o Symfony2 a partir de diferentes S.O;
Porém, caso não queira instalar via composer, basta realizar o download do arquivo zip pelo link: http://symfony.com/download?v=Symfony_Standard_Vendors_2.4.4.zip

Após realizar o donwload do Symfony2, caso tenha baixado em zip, basta extraí-lo na pasta www do apache.
Feito isso, tente acessar a seguinte URL: http://127.0.0.1/Symfony/web/app_dev.php.
Atenção: A porta de acesso pode variar; verifique qual é a porta de acesso que o apache está utilizando.
Caso ocorra tudo bem, abrirá uma tela de bem vindo (Welcome) do Symfony2.

Agora é necessário realizar a configuração do Symfony2:

  1. Clique na opção "CONFIGURE";
  2. Preencha as configurações do banco de dados que está sendo utilizado; Next step.
  3. Clique em "GENERATE" para gerar uma nova chave; Next step.

Caso ocorra tudo bem, aparecerá "Well done! Your distribution is configured!" na tela.

Erros comuns que podem ocorrer na instalação:
Erro de timezone: Caso ocorra erro de timezone, é necessário entrar no arquivo php.ini, e setar o timezone desejável.
Lib do PDO não instalada: Caso ocorra este erro, verifique se existe o PDO no arquivo php.ini; caso exista, verifique se ele está ativo, ou seja, sem o caractere ";" no ínicio.

Caso chegou até aqui, compreende-se que o Symfony2 já esteja devidamente configurado para a utilização.

Após realizar a instalação e configuração do Symfony2, agora é necessário implantar a solução proposta dentro dele.
Para isso, deve-se acessar o site no github: https://github.com/ThiagoRossetto/angularjs-symfony2, e clicar no botão: Download ZIP; ou também realizando o clone: git clone https://github.com/ThiagoRossetto/angularjs-symfony2.
Após realizar o download da solução proposta, é necessário substituir todos os arquivos presentes na solução proposta pelos arquivos dentro do Symfony2.
Atenção: A substituição não realizará modificações internas no Core do Symfony2. Isso significa que poderá ser efetuado futuras atualizações no framework.

Depois de substituir, precisa-se abrir o terminal, caminhar até a pasta da onde se encontra o Symfony2, e executar os seguintes comandos:

  • php app/console assets:install
  • php app/console doctrine:schema:update --force

Atenção(Windows): é necessário modificar alguns caminhos configurados no arquivo "src/AngularIntegration/Bundle/CoreBundle/Resources/config/services.yml", atribuindo para a propriedade "cacheFolder" o caminho "C:\fileduck", e o "debugFile" para "C:\fileduck\debug.log". Em seguida, será necessário a criação de uma pasta chamada "fileduck" dentro do "C:\".

Feito isso, basta acessar a URL: http://127.0.0.1/Symfony/web.
Caso tudo tenha sido realizado com sucesso, será adicionado um "/#/" ao final da URL, e aparecerá na tela: Seja bem vindo ao AngularJS + Symfony2 Integration.

Utilização:

Acessando um exemplo:
Após a instalação, basta acessar a URL: http://127.0.0.1/Symfony/web, e clicar no botão "Exemplo".
Depois de clicar no botão, será acrescentado o "exemplo/listar" na URL. E então, aparecerá uma tabela vazia, juntamente com dois botões: "Criar, listar".

  • Criar: Uma tela simples com um campo para preencher. Caso preencha e clique em "Save", o dado será persistido.
  • Listar: Uma tela simples com uma tabela de todas as informações persistidas.

Agora é hora de desfrutar a solução proposta!

Criar uma pequena aplicação:

  1. Criação um novo bundle (módulo) com integração AngularJS.
  2. Criação um novo contexto.
  3. Execução de comandos para atualizar o banco de dados e publicar arquivos de assets.

1) Criação um novo bundle (módulo) com integração AngularJS:

Para criar um novo bundle, basta abrir o terminal, caminhar até a pasta da onde se encontra o Symfony2, e executar o seguinte comando: php app/console angularjs:generate:bundle.

- Caso deseja visualizar todos os comandos, execute o comando: php app/console.

Após isso, haverá uma mensagem de boas vindas, juntamente campo para entrar com dados. Neste campo, é necessário digitar "AngularIntegration/Bundle/TesteBundle".
Repare que as palavras que estão em negrito são obrigatórias nesta versão.

Namespace do bundle: AngularIntegration/Bundle/TesteBundle (Enter)

A entrada acima significa : {Nome do projeto} / Bundle / {Nome do bundle}.

Após isso, deverá entrar com o tipo de dado para as configurações que deseja trabalhar neste bundle. O recomendado é o YML, por ser mais fácil a visualização das configurações.

Formato de configuração (yml, xml, php, ou annotation): yml (Enter)

Caso tudo ocorrer bem, aparecerá uma mensagem de sucesso ao criar o bundle.

2) Criação um novo contexto:

Para criar um novo contexto, basta abrir o terminal, caminhar até a pasta da onde se encontra o Symfony2, e executar o seguinte comando: php app/console angularjs:generate:context.

Após isso, haverá um campo para entrar com dados. Neste campo, é necessário digitar "AngularIntegrationTesteBundle:TesteContext".

The Context shortcut name: AngularIntegrationTesteBundle:TesteContext (Enter)

A entrada acima significa : {Nome do projeto}{Nome do Bundle} : {Nome do contexto}.

Após isso, deverá escolher qual é o tipo de dado que se irá trabalhar no contexto. No caso do contexto, o aconselhado é o annotation, pois será trabalhado com dados de entidades.

Configuration format (yml, xml, php, or annotation) [annotation]: annotation (Enter)

Após isso, deverá entrar com os campos que a entidade deste contexto possuirá. Caso terminar, basta deixar o campo vazio e pressionar enter.

Observação: Lembrando que um contexto sempre possuirá uma ou mais entidades, nunca nenhuma! Após a realização do procedimento acima, caso ocorra tudo certo, o contexto já estará disponível.

3) Execução de comandos para atualizar o banco de dados e publicar arquivos de assets:

Após realizar os passos anteriores, será necessário atualizar o banco de dados e publicar os arquivos de assets.

Para isso, basta acessar a pasta do Symfony2 e digitar os seguintes comando no terminal:

  • php app/console assets:install
  • php app/console doctrine:schema:update --force

Erros comuns que podem ocorrer na execução dos comandos:
Erro de permissão: Caso ocorra erro de permissão, será necessário dar acesso de escrita nas pastas "app/cache" e "app/logs" para o usuário que está executando o comando.

Feito isso, basta acessar a URL: http://127.0.0.1/Symfony/web/teste/#/testecontext.

Caso ocorra tudo certo, aparecerá uma tela semelhante à aquela do exemplo.
